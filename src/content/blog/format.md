この記事では、ブログ執筆に欠かせない **Markdown（マークダウン）記法** の使い方をまとめます。

---

## 1. 文字の大きさを変える
行頭に `#` を入れます。`#` の数が増えるほど見出しが小さくなります。

* `# 見出し1`：タイトル 
* `## 見出し2`：セクションの区切り
* `### 見出し3`：さらに細かい項目
* `#### 見出し4`：補足的な見出し

---

## 2. 箇条書き（リスト）
箇条書きは `*` や `-` 、番号付きは数字を使います。

### 通常のリスト
* 技術の習得
* 日々の記録
- 趣味のランニング

### 番号付きリスト
1. PCを立ち上げる
2. `npm run dev` でプレビュー
3. 記事を書いて保存する

---

## 3. 文字の強調・装飾
文章の中で目立たせたい部分に使います。

| 効果 | 書き方 | 表示例 |
| :--- | :--- | :--- |
| 太字 | `**強調**` | **強調** |
| 斜体 | `*斜体*` | *斜体* |
| 打ち消し | `~~間違い~~` | ~~間違い~~ |
| コード | `` `git push` `` | `git push` |

---

## 4. 引用と区切り線
外部からの引用や、話題をガラッと変える時に使います。

> 引用文はこのように表示されます。<br>
> 先輩エンジニアのアドバイスや、書籍の抜粋に便利です。

---

## 5. コードブロック
エンジニアブログの主役です。バッククォート3つで囲みます。

​```bash
git add .
git commit -m "update"
git push origin main
​```
